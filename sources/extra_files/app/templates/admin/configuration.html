📋 Procédure de configuration de Collabora

▶

1. Accéder au fichier de configuration

Sur le serveur hébergeant Collabora (YunoHost) :

sudo nano /etc/coolwsd/coolwsd.xml

2. Autoriser votre serveur comme hôte WOPI de confiance

Recherchez la section <storage> et ajoutez une entrée <host> avec le domaine de votre serveur orthoptie :

<storage desc="Backend storage">
    <filesystem allow="false" />
    <wopi desc="Allow/deny WOPI connections to storage">
        <host desc="Regex pattern of WOPI host" allow="true">
            {{ (cfg.wopi_base_url or wopi_base_url_default) | replace('https://', '') | replace('http://', '') }}
        </host>
    </wopi>
</storage>
⚠️ Le domaine est une expression régulière — les points doivent être échappés avec un backslash.
Exemple : dossiers\.cyps\.ynh\.fr pour dossiers.cyps.ynh.fr

3. Redémarrer Collabora

sudo systemctl restart coolwsd

4. Vérifier que Collabora est accessible

Depuis un navigateur, accédez à l'URL de découverte Collabora :

{{ (cfg.collabora_url or collabora_url_default) }}/hosting/discovery

Vous devez voir un fichier XML — si la page est vide ou en erreur, Collabora n'est pas correctement configuré.

Sur YunoHost

Si Collabora est installé via YunoHost (application Collabora Online), la configuration se fait depuis :

Webadmin → Applications → Collabora Online → Config panel

Dans le champ Allowed hosts, ajoutez le domaine de votre serveur orthoptie.
